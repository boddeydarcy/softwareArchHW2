Homework 2: Tasks for db533

Your personal task is to develop a system for a bakery, bread and their ingredients.

Task 1: Configuration [12 marks]

  1. Add all the necessary dependencies for connecting to a MySQL database and showing JSPs. [7 marks]
  2. Configure the application to use a MySQL database (username, password, etc.) and to print SQL statements on the console. [5 marks]

Task 2: Domain Classes [15 marks]

Create class co2123.hw2.model.Bakery. With the following private fields:

  1. int id
  2. String address
  3. List <Bread> breads
  4. Bread newest
Generate necessary getters and setters. [3 marks]

Create class co2123.hw2.model.Bread. With the following private fields:

  • String name
  • List <Bakery> bakeries
  • List <Ingredient> ingredients
  • Ingredient filling
Generate necessary getters and setters. [3 marks]

Create class co2123.hw2.model.Ingredient. With the following private fields:

  1. int identifier
  2. int amount
Generate necessary getters and setters. [3 marks]

Generate toString() methods for: [6 marks]

  • Ingredient which prints out the identifier and amount of this class.
  • Bread which prints out the name, ingredients, and filling of this class.
  • Bakery which prints out the id, address, breads, and newest of this class.

To format your toString(), integers should be displayed as id=X and Strings as name='X'. You can check your formatting using Task2Test.java. IntelliJ IDEA's in-built toString() generator should follow this format too.

NOTE: You must have created /listBakery, /listBread, and /listIngredient from Task 5 as well as have at least one of each object stored in your database for the toString() tests to work.

Task 3: JPA Annotations [25 marks]

Annotate the classes in co2123.hw2.model based on these statements:

  • Add the appropriate annotations so Bakery, Bread, and Ingredient become tables in your database. [3 marks]
    • Their primary keys should be id, name, and identifier respectively. [3 marks]
    • The primary keys of Bakery and Ingredient should be generated automatically. [2 marks]
  • A bakery can have many breads and a bread can belong to many bakeries. The relationship between them is bidirectional. Use mappedBy to indicate this. [2 marks]
  • A bread can have multiple ingredients but one ingredients can only belong to one bread. Use a join column as well as another annotation to indicate this. [2 marks]
  • A bread can have only one filling which belongs to one bread. [2 marks]
  • Many breads may have one newest. This newest can belong to multiple breads. [2 marks]
  • If you save a bakery, the breads in the list of breads should be saved too. [2 marks]
  • If a bread is deleted, all its ingredients should be deleted too. [2 marks].
  • If a bread is loaded, all its ingredients should be loaded too. [2 marks]

Please do not manually override the column names by using, for example, (name="NewName") on a @JoinColumn annotation.

Create CRUD repositories for each of your model classes inside the package co2123.hw2.repo called BakeryRepository, BreadRepository, and IngredientRepository. [3 marks]

Task 4: CRUD Repository Methods [20 marks]

  • Add a method in BakeryRepository which finds the bakeries by address. [5 marks]
  • Add a method in BakeryRepository which finds the bakeries by the name of their newest. [5 marks]
  • Add a method in BreadRepository which finds the breads by name. [5 marks]
  • Add a method in IngredientRepository which finds the ingredients by amount. [5 marks]

All CRUD repository methods must start with findBy (and not findAllBy, findOneBy, findBakerysBy etc.)

Task 5: Displaying Data (Model, View, and Controller) [28 marks]

  • In co2123.hw2.HW2Application, create and store at one instance of every class from co2123.hw2.model. [9 marks]
    • You will need to use the repositories you have created.
    • You must use implement CommandLineRunner and override its run method in order to create these instances.
    • The database should contain one row on each of the tables with no null or empty records.
  • Create the controller co2123.hw2.controller.ListController which has three methods: /listBakery, /listBread, and /listIngredient [5 marks]
    • They all return the JSP /src/main/webapp/WEB-INF/views/list.jsp
    • Each method should list all objects (their toString() representations) from the database on the JSP. For example, /listBakery lists all the bakeries in the database, /listBread lists all the breads, and /listIngredient lists all the ingredients. [4 marks]
    • The model attribute in each method which contains the list of bakeries/breads/ingredients should be called data.
    • Use a forEach loop on list.jsp to display the data.
  • Create the controller co2123.hw2.controller.DeleteController [1 mark]
  • For each class in package co2123.hw2.model, add a controller method to delete an instance from the database and redirect to the URL http://localhost:8080/
    • For class Bakery, the request http://localhost:8080/deleteBakery?id=5 should delete the bakery with the primary key id from the database. [3 marks]
    • For class Bread, the request http://localhost:8080/deleteBread?name=Example should delete the bread with the primary key name from the database. [3 marks]
      • HINT: You must delete all references to this bread in the bakery.
    • For class Ingredient, the request http://localhost:8080/deleteIngredient?identifier=2 should delete the ingredients with the primary key identifier from the database. [3 marks]
